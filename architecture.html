How files, people, and the agent connect

Reviewer's machine

🖥️
Author's machine

Drive sync

Drive sync
🖥️
Reviewer's machine
spec.md + spec.html
4

Reviewer annotates in Chrome

Option A
Open spec.html directly from Finder/Explorer. Works without any setup. Comments auto-save to localStorage; the in-file block updates in the browser.
Option B
CollaDoc server running as a background service → open localhost:3000/spec.html → comments auto-save directly to the file on disk. No extra steps.
🌐
spec.html
annotations: [
{ "Which method?" },
{ "USD only?" }
]
The CollaDoc server binds to 127.0.0.1 only — never exposed outside the local machine.
5

Drive syncs annotated HTML back to author

spec.html now contains the reviewer's comments. spec.md is unchanged.

Reviewer: spec.html (annotated) → Drive → Author: spec.html (annotated)
6

Agent addresses feedback

Author says to agent:
"address open feedback
in this folder"
→
1. Reads spec.html — parses open annotation threads
2. Reads spec.md — the content to edit
3. Corrections → edits spec.md silently. Decisions/opinions → embeds as > Note (author):
4. Regenerates spec.html, marks addressed threads resolved:true
↩

Loop repeats from step 3

Drive syncs updated spec.html to the reviewer. Resolved threads show in green. New round of comments can begin. spec.md is always the versioned artifact. HTML is always regenerated — never manually edited.

What lives in the folder

/project/specs/feature-xyz/
spec.md ← source of truth. edited by agent. versioned in git.
spec.html ← generated from spec.md. contains annotations. regenerated each cycle.
brief.md
brief.html ← same pattern for every MD in the folder that needs review
Annotations are stored inside each HTML file — no sidecar files. Any agent reads any HTML file directly with a standard Read tool call.

What the annotation block looks like (inside spec.html)

<!-- CollaDoc annotation store — readable by any AI -->
<script type="application/json" id="colladoc-data">
[
{
"id": "t_1715510400",
"anchor": "DLOM adjustment",
"prefix": "runs the ", // context for re-anchoring if text moves
"suffix": ", and outputs",
"comment": "Which DLOM method — Chaffe or restricted stock?",
"author": "reviewer",
"ts": "2026-05-13T09:00:00Z",
"resolved": false,
"replies": [
{ "author": "author", "text": "Chaffe. Same as Carta." }
]
}
]
</script>

Any AI reads this with a standard Read tool call. No MCP, no API, no export step.
